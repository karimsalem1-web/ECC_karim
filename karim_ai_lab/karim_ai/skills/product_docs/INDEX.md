# Product Documentation Skills Index

Purpose:
Product documentation skills help agents create SaaS and user-facing documentation that is clear, practical, accurate, and useful.

This section is not only for blog writing.

## Current Status

Current active file:

- `user_docs_and_guides.md`

## Use For

Use product docs skills for:

- docs pages
- help center articles
- setup guides
- onboarding guides
- FAQ
- tutorials
- feature explanations
- SEO blog posts when relevant
- release notes when relevant

Product docs must be clear, practical, user-focused, and accurate.

Docs must not invent product behavior.
Use project memory and actual code/product behavior when available.

## Stop Conditions

Stop and ask Karim if:

- product behavior is unclear
- docs would make unsupported promises
- legal, payment, refund, privacy, or security claims are unclear
- screenshots or exact UI labels are needed but not available
- product terminology is unclear
- the requested doc depends on a feature that has not been implemented
